---
description: Закрытие SDX-сессии по обязательному чек-листу (Closeout).
---

Ты выполняешь роль Session Manager. Твоя задача — корректно и ПОЛНОСТЬЮ закрыть активную сессию. Этот этап — самый частый источник пропусков (особенно перенос дельт в постоянные документы и удаление файлов сессии), поэтому он оформлен как обязательный чек-лист.

**Модель (ADR-009, вариант A).** Closeout проходит в ДВЕ ФАЗЫ, каждая — в своём CLI-инстансе: Фаза 1 в CLI сессии (`proj` = каталог worktree `.sdx/worktrees/<id>/`, ветка `sdx/<id>`), Фаза 2 — в ОСНОВНОМ CLI (`proj` = корень репозитория). Разведение обязательно: git запрещает checkout одной и той же ветки в двух worktree одновременно, а мёрж должен выполняться из основного дерева. Порядок исключает, чтобы основная ветка хоть на миг увидела файлы сессии — удаление оформляется коммитом `git rm -r` НА ВЕТКЕ ДО мёржа.

**Правило:** распечатай чек-лист пользователю с отметками `[ ]/[x]`, выполняй пункты по порядку и обновляй отметки. НЕ объявляй сессию закрытой, пока ВСЕ пункты не отмечены `[x]`. Если пункт неприменим к треку сессии — отметь `[x]` с пометкой `(n/a)` и поясни почему.

## Фаза 1 — в CLI СЕССИИ (`proj` = `.sdx/worktrees/<id>`, ветка `sdx/<id>`)

1. Прочитай `session_state.json`: `session_id`, `type`, `track`, `git_branch`, список артефактов.
2. Переведи `stage` в `Closeout`, запиши `[STAGE_CHANGE] Переход на этап Closeout`.
3. Пройди чек-лист закрытия, отмечая каждый пункт:

   - `[ ]` **1. Коммиты.** Рабочее дерево worktree чистое (`git status`), все изменения закоммичены на ветку `sdx/<id>`.
   - `[ ]` **2. Перенос дельт в постоянные документы.** Это ключевой пункт:
     - Бизнес-дельты (`change_note.md` или `SPEC.md` сессии) → `docs/specs/`.
     - Технические дельты (`DESIGN.md` сессии) → `docs/designs/`.
     - Пользовательская и техническая документация (README, мануалы, OpenAPI) актуализированы под внесённые изменения. При необходимости вызови субагента `tech-writer` (Task).
     - Убедись, что в постоянных доках не осталось расхождений с фактическим кодом (триада SPEC ↔ DESIGN ↔ Codebase).
     - Закоммить перенос на ветку `sdx/<id>` (это отдельный коммит от коммита удаления в п.5).
   - `[ ]` **3. Архив плана.** `PLAN.md` (если создавался) перенесён в `docs/history/plans/<session_id>.md`. Закоммить.
   - `[ ]` **4. Глобальный лог знаний.** Краткая запись об итогах сессии добавлена в `docs/history/` (что сделано, какие документы затронуты, ссылка на коммит/ветку). Закоммить.
   - `[ ]` **5. Удаление файлов сессии (коммит на ветке, ДО мёржа — вариант A).**
     ```bash
     git rm -r .claude/sessions/<session_id>
     git commit -m "sdx(<session_id>): drop session artifacts pre-merge"
     ```
     Это отдельный, явно поименованный коммит — НЕ смешивать с коммитом переноса дельт (п.2). После него HEAD-дерево ветки `sdx/<id>` больше не содержит каталог сессии (REQ-SESS-3); на диске worktree файлы тоже удалены, но каталог worktree остаётся до Фазы 2.
   - `[ ]` **6. ХЕНДОФФ на основной CLI.** Мёрж должен выполняться из ОСНОВНОГО worktree (git не позволяет checkout ветки `sdx/<id>` параллельно в двух worktree, а `main`/основная ветка чекаутится именно там). Сообщи пользователю: «Переключитесь на CLI, запущенный в корне репозитория, и выполните Фазу 2 — шаги 7–8 этого чек-листа (резолв основной ветки → мёрж → `archive-verify.sh`).» Фаза 2 выполняется как явные шаги в основном CLI; отдельного режима/флага команды для неё нет.

## Фаза 2 — в ОСНОВНОМ CLI (`proj` = корень репозитория)

   - `[ ]` **7. Резолв основной ветки и слияние.**
     ```bash
     def=$(bash .claude/sdx/hooks/lib/default-branch.sh)
     git checkout "$def"
     git merge --no-ff sdx/<session_id> -m "Merge sdx/<session_id>"
     ```
     Docs-дельты вливаются в основную ветку; каталога сессии в HEAD-дереве нет (он был удалён коммитом п.5). `--no-ff` гарантирует, что вся история ветки `sdx/<id>` достижима из merge-коммита (REQ-SESS-4).
   - `[ ]` **8. Enforcement-проверка закрытия.** Вызови:
     ```bash
     bash .claude/sdx/hooks/archive-verify.sh <session_id>
     ```
     Скрипт резолвит основную ветку через `lib/default-branch.sh` (без хардкода `main`) и выполняет инвариант 1 (чистое дерево), 5 (ветка слита в основную) и 6 (каталог сессии не tracked в дереве основной ветки — детектор пропуска варианта A), а затем сам освобождает worktree (`git worktree remove --force .sdx/worktrees/<session_id>`, REQ-WT-5) и удаляет слитую ветку (`git branch -d`). Ненулевой код выхода → Closeout НЕ завершён: устрани FAIL (обычно — пропущенный п.5) и повтори.
   - `[ ]` **9. Worktree и файлы сессии удалены.** Выполнено скриптом `archive-verify.sh` (п.8) — enforced. `git worktree list` не должен содержать записи закрытой сессии; каталог `.claude/sessions/<session_id>/` отсутствует в основном дереве.
   - `[ ]` **10. Финальная запись.** Факт закрытия фиксируется в глобальном логе (п.4, уже выполнен в Фазе 1) — `session.log` отдельной записи не требует, так как он был удалён вместе с каталогом сессии в п.5/п.8.

После того как все пункты отмечены `[x]`, выведи итоговую сводку закрытия: что перенесено, куда слито, что удалено.
