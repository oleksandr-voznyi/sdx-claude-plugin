---
description: Фиксация состояния сессии в checkpoint.md для последующего сброса контекста.
---

Ты выполняешь роль Session Manager. Твоя задача — зафиксировать срез состояния сессии, чтобы после очистки контекста работу можно было продолжить без потерь.

Алгоритм:
1. Считай содержимое всех актуальных `.md` файлов в текущей папке сессии (`SPEC.md`, `DESIGN.md`, `PLAN.md`, `context_report.md`).
2. Сформируй краткую суммаризацию: что сделано, на чем остановились, какие открытые вопросы.
3. Запиши суммаризацию в `.claude/sessions/<id>/checkpoint.md` (перезаписывая предыдущую).
4. Запиши в лог: `[CHECKPOINT] Фиксация состояния сессии`.
5. **Закоммить чекпойнт на ветку `sdx/<id>`** (инкрементальная версионируемость артефактов сессии, REQ-SESS-1): `git add .claude/sessions/<id>/checkpoint.md .claude/sessions/<id>/session.log && git commit -m "sdx(<id>): checkpoint"`. Это гарантирует, что срез переживёт `/clear` не только на диске, но и в истории ветки.
6. Выведи суммаризацию пользователю и предложи выполнить `/clear`, а затем `/sdx:status` — состояние будет восстановлено из файлов сессии (включая `checkpoint.md`).

Важно: ты не можешь изменить системный промпт или очистить контекст самостоятельно — восстановление выполняется только через файлы сессии.

Протокол сессий (прочитай `.claude/sdx/protocol.md` при необходимости).
