description: Экспорт функциональности в переносимый SDX-бандл (read-only).
---

Ты выполняешь роль Session Manager. Твоя задача — извлечь функциональность из текущего проекта-источника и упаковать её в переносимый бандл. Это **read-only** операция: проект-источник НЕ модифицируется, отдельная сессия не создаётся.

Аргументы: $ARGUMENTS
(Ожидается: `<feature>` — что экспортируем; опционально `--session <id>`, если фича соответствует прошлой сессии)

Алгоритм:
1. Определи границу фичи: по аргументу/описанию пользователя либо по артефактам указанной прошлой сессии. При неоднозначности — уточни у пользователя, не угадывай.
2. **Извлечение замыкания.** Вызови субагента `architect` (Task) в режиме извлечения: посчитать полное замыкание фичи —
   - требования из `SPEC.md`, относящиеся к фиче (Delta-срез);
   - компоненты, схемы данных, API из `DESIGN.md`;
   - файлы кода фичи и связанные тесты;
   - **замыкание зависимостей**: внешние библиотеки с версиями и внутренние модули, на которые фича опирается;
   - допущения о среде и точки интеграции (роуты, DI, entrypoints).
   Передавай субагенту пути и задачу, а не содержимое файлов (см. протокол).
3. **Сборка бандла** в `.sdx/bundles/<feature>/` (см. формат в протоколе):
   `manifest.yaml` (источник, версия SDX, имя фичи, дата, исходная сессия при наличии), `spec.delta.md`, `design.slice.md`, `code/`, `tests/`, `deps.yaml`, `assumptions.md`, `integration_points.md`.
4. Самопроверка полноты: каждое требование среза прослеживается в коде/тестах бандла; все зависимости, упомянутые в коде, попали в `deps.yaml`. Несоответствия выведи пользователю.
5. Выдай путь к бандлу и краткую сводку (состав, число файлов, ключевые зависимости/допущения) для переноса в целевой проект через `/sdx:import`.

Протокол сессий, формат бандла и поток import: @.claude/sdx/protocol.md
