---
description: Просмотр статуса текущей активной сессии SDX.
---

Ты выполняешь роль Session Manager. Твоя задача — предоставить отчет о текущем состоянии активной сессии. Это **чисто читающая** операция: НЕ пиши в лог и не меняй состояние (просмотр статуса не должен мутировать сессию).

Алгоритм:
1. Определи текущую сессию по активной git-ветке (`git branch --show-current`, ветки вида `sdx/<id>`) или по содержимому `.claude/sessions/`.
2. Считай `.claude/sessions/<id>/session_state.json`.
3. Выведи пользователю:
   - **Session ID**, **Track**, **Current Stage**, **Status**.
   - **Artifacts**: текущий состав файлов сессии — из `git status`/листинга папки сессии (не из лога).
   - **Progress**: % выполнения из `PLAN.md` (греп чек-боксов `- [ ]`/`- [x]`), если он есть.
4. **(факультативно) Активные сессии.** Дополнительно выведи листинг всех активных сессий в проекте через `git worktree list` — каждая запись `.sdx/worktrees/<id>` (ветка `sdx/<id>`) соответствует отдельной активной сессии. Это чисто читающий шаг, не влияющий на текущую сессию.

Протокол сессий (прочитай `.claude/sdx/protocol.md` при необходимости).
