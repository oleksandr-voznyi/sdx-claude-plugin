Ты выполняешь роль Session Manager. Твоя задача — переключить рабочее окружение на другую сессию.

Аргументы: $1
(Ожидается: <session_id>)

В worktree-модели (см. protocol.md, «Модель сессии = worktree = ветка») переключение сессий НЕ является git-операцией над текущим рабочим деревом — каждая сессия физически изолирована в собственном каталоге worktree. `/sdx:switch` — чисто читающая/навигационная операция.

Алгоритм:
1. Выполни `git worktree list --porcelain`, найди запись с `branch refs/heads/sdx/$1` и извлеки её путь (`worktree <path>`).
2. Если запись не найдена → сообщи ошибку: «сессия `$1` не активна (нет worktree) — запустите `/sdx:start` или сессия уже заархивирована» — и останови выполнение.
3. **[REMOVED] слепой авто-коммит.** НИКАКИХ `git add -A`, `git commit`, `git checkout` не выполняется (REQ-WT-2) — переключение больше не мутирует git-состояние ни текущей, ни целевой ветки.
4. Инструкция пользователю: «Откройте/запустите CLI в каталоге `<путь-из-шага-1>` — это и есть переключение на сессию `$1`. Текущее рабочее дерево не затрагивается.»
5. Подтверди пользователю найденный путь и трек/этап сессии (по `session_state.json` внутри найденного каталога, если нужно предпросмотреть, — операция чтения, без `git checkout`).
