# Change Note — fw-auto-gates-20260719

**Задача:** E4 (авторежим гейтов по запросу пользователя) + C3 (обязательная независимая
проверка patch-трека). Источник: `docs/audit-2026-07-01-recommendations.md` §E4, §C3
(фидбек пользователя 2026-07-19).

---

## Discovery (инлайн, standard)

**Контекст.** Два слоя контроля SDX: детерминированные хуки (stage-gate, stop-gate,
prod-guard, archive-verify) и прозаические гейты команд (аппрувы артефактов, scope-check
на `/sdx:next`, квитирование WARN). E4 смягчает ТОЛЬКО прозаический слой; хук-логика
не меняется (ноль правок `sdx/hooks/**`).

**Затрагиваемые файлы:** `sdx/protocol.md`, `commands/{start,next,verify,archive,retrack}.md`,
`docs/DECISIONS.md` (новый ADR-014), `CLAUDE.md` (§3 patch-трек, §4 WARN — уточнения).
Все правки — `.md`-проза; stage-gate их не блокирует на любой стадии.

**Ограничения (не смягчаются авторежимом):**
- Коллизии триады (CLAUDE.md §4) — стоп без исключений.
- FAIL верификации / красный тест-пол stop-gate — блокируют как блокировали.
- `/sdx:manual` интерактивен по определению (ADR-006) — авторежим его не автоматизирует.

**Согласованные развилки (пользователь, 2026-07-19, Discovery-гейт):**
1. C3: лёгкий fresh-eyes на patch-треке обязателен **всегда** (не только в авторежиме).
2. WARN в авторежиме: **авто-квитирование с дисклоузом** первым блоком перед Closeout
   (правка инварианта ADR-004 новым ADR-014).

---

## Решения (Change: бизнес + технические, слитно)

### E4 — авторежим гейтов (opt-in)

1. **Поле состояния.** `gate_mode: "interactive" | "auto"` в `session_state.json`;
   default `interactive`. Схема — в `protocol.md`.
2. **Включение.** `/sdx:start --auto` (сразу с гипотезой трека) ИЛИ предложение оркестратора
   на Discovery/Change-гейте `/sdx:next`. Решение per-session. Возврат в `interactive` —
   в любой момент по слову пользователя или автоматически (п.6).
3. **Поведение при `auto`.** Прозаические подтверждения (аппрувы Spec/Design/Plan/change_note,
   переходы `/sdx:next`, несущественные уточнения по ходу) принимают дефолт без остановки.
   Каждый принятый дефолт — одна строка в файл сессии `auto_decisions.md`
   (формат: `развилка → выбранный дефолт → почему несущественно`). Файл трек-независим
   (работает и на full, где change_note.md отсутствует), версионируется как артефакт сессии.
4. **Стоп-рубрика** (авторежим ОБЯЗАН остановиться; фиксируется в `protocol.md`):
   - коллизии триады SPEC↔DESIGN↔Code;
   - FAIL верификации, красный тест-пол;
   - изменения внешних контрактов: публичные API, схемы данных, форматы файлов, миграции;
   - деструктивные/необратимые операции; территория prod-guard;
   - эскалация трека (см. п.6);
   - шаги ручного тестирования (`/sdx:manual`).
   Критерий по умолчанию: цена отката ошибочного дефолта выше цены остановки → стоп.
5. **Дисклоуз (единственная обязательная точка подтверждения).** Новый пункт чек-листа
   `/sdx:archive` (на ветке, до мёржа): вывести `auto_decisions.md` одним блоком —
   **автоквитированные WARN первыми**, затем прочие дефолты — и получить одно явное
   подтверждение пользователя. Не устроило → `/sdx:backtrack`/`/sdx:retrack` штатно.
6. **Эскалация трека в авто.** `/sdx:retrack` вверх (patch→standard→full) сбрасывает
   `gate_mode → interactive`: масштаб изменился, исходное решение «задача тривиальна»
   недействительно. Деэскалация режим не трогает.
7. **ADR-014.** Правит инвариант ADR-004 «WARN требует явного квитирования пользователем»:
   в `gate_mode=auto` квитирование выполняется **отложенно** — WARN автоквитируются и
   предъявляются первым блоком дисклоуза на Closeout (п.5). Прецедент пересмотра — ADR-008.

### C3 — обязательный лёгкий fresh-eyes на patch

8. **Verification на patch-треке перестаёт быть опциональной.** Активные этапы patch:
   `Execution → Verification (лёгкая, обязательная) → Closeout`. Независимо от `gate_mode`.
9. **«Лёгкий» прогон** = fresh-eyes `reviewer` через Task (контракт изоляции ADR-004
   без изменений), суженный объём: diff поставки + `change_note.md` (оси корректность
   и когерентность; полнота — в границах change_note). FAIL блокирует Closeout;
   WARN — по общим правилам (interactive: явный квит; auto: дисклоуз, п.5).
   Корректность-исполнением (`qa`) на patch по-прежнему не требуется — фикс-тест
   и stop-gate покрывают.
10. **Гейт archive.** `/sdx:archive` на patch проверяет наличие `verification_report.md`
    без FAIL до начала чек-листа; отсутствует → стоп с подсказкой `/sdx:verify`.

### Карта правок (для Execution)

| Файл | Правка |
|------|--------|
| `sdx/protocol.md` | схема состояния (+`gate_mode`); новый раздел «Авторежим гейтов» (пп.2–6); таблица треков (patch: Verification обязательна, лёгкая); §Верификация (масштаб по треку: patch — обязательный суженный); §Closeout (+пункт дисклоуза в чек-лист) |
| `commands/start.md` | флаг `--auto`; инициализация `gate_mode` в state |
| `commands/next.md` | предложение авторежима на Discovery/Change-гейте; поведение гейтов при `auto` (дефолт + запись в `auto_decisions.md`); ссылка на стоп-рубрику |
| `commands/verify.md` | patch: «по запросу» → «обязательный суженный прогон»; WARN при `auto` — авто-квит с записью |
| `commands/archive.md` | новый пункт чек-листа «дисклоуз auto_decisions (WARN первыми), одно подтверждение»; гейт п.10 |
| `commands/retrack.md` | эскалация сбрасывает `gate_mode → interactive` |
| `docs/DECISIONS.md` | ADR-014 (авторежим + отложенное квитирование WARN + обязательная лёгкая верификация patch) |
| `CLAUDE.md` | §3: `(Verification)` → обязательная лёгкая на patch; §4: уточнение про отложенное квитирование в авто |

**Вне скоупа:** изменения хук-скриптов и их тестов; тир `reviewer` (остаётся `opus`,
инвариант ADR-008); автоматизация `/sdx:manual`.
