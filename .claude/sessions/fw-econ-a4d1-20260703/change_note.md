## Discovery (инлайн)

### A4 — stop-gate гоняет полный тест-сьют на каждом завершении хода
- `stop-gate.sh` на стадиях `Execution|Verification` запускает verify-команду при **каждом**
  `Stop`. Длинная Execution из десятков ходов = десятки прогонов, даже когда код не менялся.
- Существующие эфемерные файлы гейта: `.stopgate.count` (loop-guard), `.stopgate.out` (вывод).
  Кэша «последнего зелёного состояния дерева» нет.
- Все `.stopgate.*` игнорируются точечным паттерном в `.gitignore` (REQ-SESS-2) — новый
  `.stopgate.ok` попадает под тот же паттерн, версионировать его не нужно.

### D1 — `@protocol.md` инжектится каждой командой
- Полный `@.claude/sdx/protocol.md` (~210 строк) подтягивается 11 командами: `backtrack`,
  `checkpoint`, `retrack`, `start`, `export`, `archive`, `import`, `manual`, `status`, `next`,
  `switch`, `verify`.
- «Тонким» командам (короткая механика, не нужен весь протокол целиком) инжект избыточен.

## Change (решения — бизнес + технические, слитно)

### A4 — кэш зелёного прогона по состоянию дерева
**Решение.** Перед запуском verify-команды вычислять отпечаток рабочего дерева и, если он
совпадает с сохранённым отпечатком последнего зелёного прогона, выходить `exit 0` без прогона.
- Отпечаток = `git rev-parse HEAD` + хэш `git status --porcelain` (учитывает и коммиты, и
  незакоммиченные изменения). Хранить в `.claude/sessions/<id>/.stopgate.ok`.
- Кэш пишется **только** при зелёном прогоне (рядом со сбросом loop-guard `.stopgate.count`).
- Проверка кэша размещается **после** резолва verify-команды и **до** инкремента loop-guard —
  чтобы no-op по неизменному дереву не крутил счётчик и не запускал сам прогон.
- Инвалидируется автоматически: любое изменение дерева → другой отпечаток → прогон повторяется.
- `SDX_STOP_GATE=1` (headless/форс) кэш **обходит** — форсированный прогон должен быть честным.

**Затрагивает:** `.claude/sdx/hooks/stop-gate.sh`, `.claude/sdx/hooks/test-stop-gate.sh`
(+ тест: неизменное дерево после зелёного → no-op; изменённое дерево → повторный прогон).

### D1 — тонкие команды ссылаются на протокол текстом
**Решение.** Убрать `@`-инжект полного `protocol.md` у «тонких» команд, заменив на текстовую
ссылку «прочитай `.claude/sdx/protocol.md` при необходимости». `@`-инжект сохранить у команд,
которым протокол нужен целиком по ходу исполнения.
- **Текстовая ссылка (снять `@`):** `status`, `checkpoint`, `switch`, `backtrack` — короткая
  детерминированная механика, полный протокол по ходу не разворачивают.
- **Сохранить `@`-инжект:** `start`, `next`, `archive`, `verify` (нужны профили флоу / контракт
  закрытия / оси верификации целиком), а также `retrack`, `export`, `import`, `manual`
  (нужны таблица треков / формат бандла / поток import / оси верификации).

**Затрагивает:** `.claude/commands/sdx/{status,checkpoint,switch,backtrack}.md`.
