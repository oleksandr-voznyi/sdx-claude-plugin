# Change Note: отказ от worktree-модели сессий — сессия = ветка в основном рабочем дереве

## Проблема (запрос пользователя, 2026-07-05)
Worktree-модель (ADR-009) сделала процесс операционно тяжёлым: `/sdx:start` заканчивается
хендоффом «откройте новый CLI в `.sdx/worktrees/<id>/`», Closeout требует ДВУХ CLI-инстансов
(Фаза 1 в worktree, Фаза 2 в корне). Пользователь не может продолжать разработку в том же CLI —
это ломает непрерывность работы и обесценивает оркестрацию. Практика показала: цена физической
изоляции (REQ-WT-3) выше её пользы.

## Решение (бизнес + техника, слитно)
Перейти на модель **«сессия = ветка `sdx/<id>`, checkout в основном рабочем дереве, один CLI»**:

1. **Что сохраняется из ADR-009 (не откатывается):**
   - Версионирование артефактов сессии на ветке `sdx/<id>` инкрементальными коммитами (REQ-SESS-1/2, ADR-005).
   - Вариант A: `git rm -r .claude/sessions/<id>` коммитом НА ВЕТКЕ до мёржа; `merge --no-ff` (REQ-SESS-3/4).
   - Автоопределение основной ветки (ADR-010) и все инварианты archive-verify (1/5/6).
   - Запрет слепого авто-коммита в `/sdx:switch` (ядро REQ-WT-2).
2. **Что меняется:**
   - `/sdx:start`: вместо `git worktree add` — `git checkout -b sdx/<id>` в текущем дереве
     (требуется чистое дерево). Хендофф на другой CLI УДАЛЁН — работа продолжается тем же CLI.
   - `/sdx:switch`: снова git-операция — `git checkout sdx/<id>` при чистом дереве; при грязном —
     стоп с инструкцией закоммитить в текущую сессию (авто-коммита нет).
   - `/sdx:archive`: одна фаза, один CLI — весь чек-лист (включая мёрж и `archive-verify.sh`)
     выполняется последовательно в том же CLI.
   - `/sdx:status`: листинг активных сессий через `git branch --list 'sdx/*'` вместо `git worktree list`.
   - Хуки: правки НЕ требуются — `stage-gate.sh`/`stop-gate.sh` уже резолвят сессию по имени
     текущей ветки; `$CLAUDE_PROJECT_DIR` = корень репозитория и корректен по построению
     (CLI один, запущен в корне). `archive-verify.sh` worktree-агностичен: блок
     `git worktree remove` условный (`[ -n "$wt" ]`) и деградирует в no-op — оставлен как
     совместимость для добивания legacy-worktree.
   - `.gitignore`: паттерн `.sdx/worktrees/` сохраняется (legacy-совместимость, безвреден).
3. **Что теряется (осознанный трейд-офф):**
   - REQ-WT-3 (физическая изоляция параллельных «грязных» сессий) — снимается. Параллельные
     сессии остаются возможны как ветки, но переключение требует чистого дерева
     (инкрементальные коммиты — и так норма протокола, REQ-SESS-1).
   - REQ-WT-1/4/5 — теряют предмет (worktree больше не создаётся).

## Обоснование выбора
Худший режим worktree-модели — операционный: два CLI на закрытие одной сессии и обязательный
перезапуск CLI на старте. Худший режим in-place-модели — потеря незакоммиченных правок при
недисциплинированном переключении, что купируется (а) требованием чистого дерева на
`checkout`, (б) нормой инкрементальных коммитов. Enforcement-слой инвариантен к модели —
стоимость перехода почти нулевая (правки только в prose-командах и протоколе).

## Затронутые файлы
- `.claude/sdx/protocol.md` — секция «Модель сессии», контракт Closeout (одна фаза), описание archive-verify.
- `.claude/commands/sdx/start.md` — шаги 3–10 (checkout вместо worktree, без хендоффа).
- `.claude/commands/sdx/switch.md` — checkout-семантика с гардом чистого дерева.
- `.claude/commands/sdx/archive.md` — однофазный чек-лист.
- `.claude/commands/sdx/status.md` — листинг сессий по веткам.
- `.claude/commands/sdx/init.md` — текст про модель и `.gitignore`.
- `.claude/commands/sdx/verify.md` — формулировка причины исключения каталога сессии из diff.
- `CLAUDE.md` §6 — описание `.claude/sessions/<id>/`.
- `docs/DECISIONS.md` — новый ADR-012 (пересматривает worktree-часть ADR-009).
- `.claude/sdx/hooks/archive-verify.sh` — только комментарий заголовка (логика не меняется).
