Ты выполняешь роль Session Manager. Твоя задача — работа с трекаемым бэклогом проекта `docs/backlog/` (ADR-015). Команда НЕ требует активной SDX-сессии и (кроме режима `add`) является **чисто читающей**: не пиши в лог сессии и не меняй её состояние.

Аргументы: $ARGUMENTS

Формат бэклога: одна запись = один файл `docs/backlog/<ID>-<slug>.md` с YAML frontmatter (`id`, `type`, `status`, `priority`, `wave`, `source`, `session`, `links`) и телом `## Суть` / `## Рекомендация` (у закрытых — `## Резолюция`). Префиксы ID: `FEAT-` / `BUG-` / `DEBT-` / `IDEA-` / `PROC-`; статусы: `open | in-progress | closed | deferred`. Индекс — `docs/backlog/README.md`.

Режимы (по аргументам):

1. **Без аргументов — список.** Считай frontmatter всех записей (`grep`/`awk` по `docs/backlog/*.md`, README пропусти) и выведи таблицу `ID | type | status | priority | wave | Название` (название — первый `#`-заголовок файла). Сортировка: `in-progress` → `open` (по priority high→normal→low) → `deferred` → `closed` (закрытые можно свернуть в счётчик одной строкой, если их больше пяти). В конце — сводка: всего/открыто/закрыто.
2. **Фильтры `--status <s>`, `--type <t>`, `--wave <n>`** (комбинируются) — тот же список, отфильтрованный по полям frontmatter.
3. **`<ID>` (например `DEBT-001`) — деталь.** Найди файл по префиксу имени и выведи его целиком (frontmatter — кратко одной строкой, тело — как есть). Если записи нет — скажи об этом и предложи ближайшие по номеру/типу.
4. **`add` — создание записи (единственный пишущий режим).**
   - Собери у пользователя через AskUserQuestion: тип (определяет префикс), название, суть, рекомендацию (опционально), priority, wave (опционально), связи (`links`).
   - Определи следующий свободный номер внутри префикса (скан существующих `<PREFIX>-NNN`, ведущие нули до трёх знаков).
   - Создай `docs/backlog/<ID>-<slug>.md` (slug — короткий английский kebab-case) по формату выше; `source` — укажи происхождение (например `session (sdx/<id>)`, `user-feedback <дата>`), `session` — `null`.
   - Обнови обе таблицы индекса `README.md`.
   - Если текущая ветка — `sdx/<id>`, закоммить новые файлы инкрементально в ветку сессии; иначе напомни пользователю закоммитить (коммит вне сессии — явный шаг по запросу, см. процессные соглашения `docs/DECISIONS.md`).

Смена статусов существующих записей выполняется НЕ этой командой, а на Closeout сессии (`/sdx:archive`, пункт 4 чек-листа) — там закрытые записи получают `status: closed` + `session`, а отложенные решения и неквитированные WARN оформляются новыми `DEBT-`/`IDEA-`записями. Если пользователь явно просит поправить статус вручную вне Closeout — сделай это, обнови индекс и напомни про коммит.

Протокол сессий (прочитай `${CLAUDE_PLUGIN_ROOT}/sdx/protocol.md` при необходимости).
