---
argument-hint: --to <stage>
description: Возврат на предыдущий этап SDX для внесения правок.
---

Ты выполняешь роль Session Manager. Твоя задача — безопасно откатить сессию на указанный этап.

Аргументы: $ARGUMENTS
(Ожидается: --to <stage>)

Алгоритм:
1. Вызови единственный писатель `stage`:
   ```bash
   bash "${CLAUDE_PLUGIN_ROOT}/sdx/hooks/sdx-stage.sh" backtrack "<session_id>" "<stage>"
   ```
   Скрипт сам проверяет, что `<stage>` — распознанное имя этапа протокола, активное в текущем треке, и не позже текущего этапа; при успехе переводит `stage`, атомарно пишет `[STAGE_CHANGE]` в лог и помечает устаревшие артефакты (все этапы строго после `<stage>` в порядке трека, БЕЗ верхней границы — до конца активных этапов трека, а не только до текущего этапа: артефакт этапа, который позже текущего, но всё же оставил файл на диске от предыдущего цикла, тоже помечается) HTML-баннером `<!-- SDX-OUTDATED -->` первой строкой файла — без переименования/зануления/бэкапа.
2. **Код `0`** (включая no-op `OK no-op <stage>`, если `<stage>` совпал с текущим этапом — файл не тронут): закоммить смену состояния на ветку `sdx/<id>` вместе с помеченными файлами (если stdout действительно содержит строки `OUTDATED: <path>`):
   ```bash
   git add .claude/sessions/<id>/session_state.json .claude/sessions/<id>/session.log <помеченные файлы> && git commit -m "sdx(<id>): backtrack -> <stage>"
   ```
   Сообщи пользователю ровно те пути, что перечислены строками `OUTDATED: <path>` в stdout, как файлы, требующие актуализации — не догадывайся о них самостоятельно. Для no-op коммитить нечего.
3. **Код `1`** (имя не распознано / этап не активен в текущем треке / целевой этап позже текущего — не откат): сообщи пользователю текст stderr дословно (для случая «не активен в треке» он уже направляет на `/sdx:retrack`). `stage` не менялся, коммитить нечего.
4. **Код `2`** (нет `session_state.json` / нет `jq` / состояние повреждено): сообщи пользователю текст stderr дословно и остановись.

Протокол сессий (прочитай `${CLAUDE_PLUGIN_ROOT}/sdx/protocol.md` при необходимости).
