description: Переключение между активными SDX сессиями.

Ты выполняешь роль Session Manager. Твоя задача — переключить рабочее окружение на другую сессию.

Аргументы: $1
(Ожидается: <session_id>)

В модели «сессия = ветка в основном дереве» (ADR-012, см. protocol.md) переключение сессии — это обычный `git checkout` ветки `sdx/<id>` в текущем рабочем дереве, в том же CLI. Слепой авто-коммит ЗАПРЕЩЁН (ядро REQ-WT-2): НИКАКИХ `git add -A && git commit` от имени пользователя.

Алгоритм:
1. Проверь, что ветка существует: `git rev-parse --verify sdx/$1`. Если нет → сообщи ошибку: «сессия `$1` не активна (нет ветки `sdx/$1`) — запустите `/sdx:start` или сессия уже заархивирована» — и останови выполнение.
2. **Гард чистого дерева.** Если `git status --porcelain` непуст → остановись и сообщи: «рабочее дерево содержит незакоммиченные изменения — закоммитьте их в текущую сессию (инкрементальные коммиты — норма протокола, REQ-SESS-1) или стэшните, затем повторите `/sdx:switch`». Если текущая сессия — трека `vibe` на стадии `Prototype`, коммит НЕ является требованием (REQ-VIBE-3/9): штатный путь переключения — `git stash` (с последующим `git stash pop` по возвращении), либо сначала пройти гейт `/sdx:proto`. Авто-коммит не выполняй.
3. Переключись: `git checkout sdx/$1`.
4. Прочитай `.claude/sessions/$1/session_state.json`, подтверди пользователю трек/этап, запиши в `session.log` строку `[START] Переключение на сессию $1`.

Протокол сессий (прочитай `${CLAUDE_PLUGIN_ROOT}/sdx/protocol.md` при необходимости).
