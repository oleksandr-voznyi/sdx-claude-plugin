# BUG-002. Prod-guard fail-closed блокирует весь Bash даже без сконфигурированной защиты

## Суть
`prod-guard.sh` проверяет `jq` до проверки наличия `prod-guard.conf`. На проекте
без прод-среды (conf отсутствует/пустой = «защиты нет по договорённости») отсутствие `jq`
превращается в блокировку каждой Bash-команды, включая `ls`. Fail-closed оправдан только
когда защита реально сконфигурирована.

## Рекомендация
Поднять проверку conf (`[ -f ... ] || exit 0`; лучше — «есть ли хоть один
не-комментарий») выше jq-проверки — она jq не требует. Fail-closed остаётся для проектов
с заполненным conf. Добавить тест.

## Резолюция
Закрыто (сессия `fw-enforce-a1a2-20260703`, 2026-07-03) — проверка conf (с has_pattern)
поднята выше jq; fail-closed только при сконфигурированной защите.
