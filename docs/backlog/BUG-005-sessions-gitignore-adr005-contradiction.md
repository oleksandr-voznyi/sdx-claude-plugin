# BUG-005. Противоречие: ADR-005 требует инкрементальных коммитов сессии ↔ `.claude/sessions/` в `.gitignore`

## Суть
ADR-005 предписывает: «Артефакты коммитятся в ветку сессии инкрементально
(поддерживает вывод истории из git и Closeout)» (`docs/DECISIONS.md`, стр. 47). Но
`.claude/sessions/` внесён в `.gitignore` (стр. 2), т.е. файлы сессии физически не могут
быть закоммичены — они невидимы для git. Ломается заявленное основание ADR-005 (вывод состава
артефактов и истории из git) и Closeout-шаг «коммиты в ветку сессии». Сейчас сессии живут
только на локальном уровне и не переносятся между машинами/участниками команды.

## Рекомендация
Проработать в рамках рефакторинга работы с сессиями: перенести сессии с
локального уровня на уровень видимости командой (снять `.claude/sessions/` из `.gitignore`
либо выделить трекаемый подкаталог для артефактов, оставив в ignore только эфемерное).
Согласовать с моделью параллельных сессий из C2 (git worktree): одна сессия = свой worktree,
артефакты версионируются в ветке `sdx/<id>`.
