# BUG-007. `/sdx:init` создаёт каталоги `docs/history/{retro,postmortem,intake}/` без файлов-заглушек

## Суть
`commands/init.md` добавляет три новых каталога в список создаваемых при инициализации
проекта, но не кладёт в них файл-заглушку. Git не хранит пустые каталоги — в свежем
проекте они не переживут первый коммит, и первая же `retro`/`postmortem`/`intake`-сессия
на Closeout будет переносить документ разбора в несуществующий путь.

В мета-репозитории SDX проблема не проявляется: там каталоги созданы задачей T-21 вместе
с `README.md`-заглушками, поэтому дефект виден только на новом проекте.

## Рекомендация
Привести `commands/init.md` в соответствие с фактической раскладкой мета-репозитория:
создавать в каждом из трёх каталогов `README.md` (одна фраза о назначении + ссылка на
ADR-017), как это сделано в самом SDX. Альтернатива — `.gitkeep`, но тогда разойдётся
с уже существующей раскладкой.
