# DEBT-004. Мета-проект имеет тест-сьют, но не «доедает свой корм»

## Суть
ADR-4 и SPEC исходят из «у мета-проекта нет тестов → stop-gate no-op». Уже
неправда: в `.claude/sdx/hooks/` лежит 31 юнит-тест (`test-*.sh`), и ничто их не запускает
автоматически.

## Рекомендация
Создать исполняемый `.claude/sdx/verify-cmd.sh` мета-проекта, запускающий
`test-*.sh` — stop-gate станет активен на будущих SDX-сессиях фреймворка (dogfooding),
регресс хуков будет ловиться детерминированно. Обновить примечание в `verify.md`
(«в мета-проекте stop-gate прозрачен» — устареет) и ADR-4.
