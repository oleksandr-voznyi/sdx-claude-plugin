# DEBT-004. Мета-проект имеет тест-сьют, но не «доедает свой корм»

## Суть
ADR-4 и SPEC исходят из «у мета-проекта нет тестов → stop-gate no-op». Уже
неправда: в `.claude/sdx/hooks/` лежит 31 юнит-тест (`test-*.sh`), и ничто их не запускает
автоматически.

## Рекомендация
Создать исполняемый `.claude/sdx/verify-cmd.sh` мета-проекта, запускающий
`test-*.sh` — stop-gate станет активен на будущих SDX-сессиях фреймворка (dogfooding),
регресс хуков будет ловиться детерминированно. Обновить примечание в `verify.md`
(«в мета-проекте stop-gate прозрачен» — устареет) и ADR-4.

## Резолюция

Закрыто сессией `fw-dogfood-verifycmd-20260720` (2026-07-20). Создан исполняемый
`.claude/sdx/verify-cmd.sh` (глоб по `sdx/hooks/test-*.sh`, guard пустого глоба, exit 1 при
любом красном сьюте); stop-gate подтверждён активным end-to-end на самой сессии закрытия.
Примечание в `commands/verify.md` переформулировано генерически; датированные поправки
DEBT-004 внесены в ADR-4 дизайна и допущение №4 SPEC Фазы 1 (no-op-деградация остаётся
спроектированным поведением для проектов без тест-команды). Уточнение фактов записи: тестов
на момент закрытия 56 (5 сьютов), путь — `sdx/hooks/` (не `.claude/sdx/hooks/` — запись
ссылалась на до-плагинную раскладку). Отсутствие автотеста на сам раннер — DEBT-013.
