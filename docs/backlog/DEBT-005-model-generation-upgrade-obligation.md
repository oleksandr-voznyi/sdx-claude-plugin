# DEBT-005. Сработало обязательство обновить раскладку моделей на новое поколение

## Суть
DESIGN сознательно пиннит полные ID (`claude-sonnet-4-6`, `claude-opus-4-8`,
`claude-haiku-4-5`); бандл фиксирует обязательство «при смене поколения обновить раскладку».
Поколение сменилось: актуальная линейка — семейство Claude 5 (`claude-sonnet-5` и выше).
Шесть агентов работают на Sonnet 4.6 при доступном Sonnet 5.

## Рекомендация
Отдельная сессия (standard): рабочие роли → `claude-sonnet-5`; для
`reviewer` взвесить Opus 4.8 vs Sonnet 5 (новое поколение может давать сравнимое суждение
дешевле). Альтернатива навсегда: перейти на алиасы (`sonnet`, `opus`, `haiku`) — схема
поддерживает — и убрать класс проблемы «пин протух», приняв трейд-офф (меньше контроля
над поколением). Решение зафиксировать в ADR.

## Резолюция
Закрыто (сессия `fw-model-aliases-20260702`, 2026-07-02) — переход на алиасы тиров, ADR-008.
