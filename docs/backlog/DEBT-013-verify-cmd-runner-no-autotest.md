# DEBT-013. У раннера `.claude/sdx/verify-cmd.sh` нет собственного автотеста

## Суть

Раннер `verify-cmd.sh` мета-проекта (DEBT-004) — новый исполняемый артефакт с собственной
логикой (агрегация exit-кодов сьютов, guard пустого глоба), но постоянного автотеста на него
нет. Red-suite и empty-glob поведение проверены вручную в изолированной копии только в сессии
закрытия DEBT-004; будущий регресс (например, при рефакторинге раннера) обнаружится лишь
вручную.

## Рекомендация

Добавить `sdx/hooks/test-verify-cmd.sh` по аналогии с остальными сьютами: кейсы — все сьюты
зелёные → exit 0 и сводка N/N; один красный сьют → остальные всё равно прогоняются, exit 1;
пустой глоб → громкое сообщение в stderr, exit 1. Благодаря глобу в самом раннере новый сьют
подхватится `verify-cmd.sh` автоматически.
