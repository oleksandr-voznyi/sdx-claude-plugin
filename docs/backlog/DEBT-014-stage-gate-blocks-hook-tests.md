# DEBT-014. stage-gate на Verification не пускает тесты хуков — конвенция репозитория не совпадает с allow-паттерном

## Суть

`sdx/hooks/stage-gate.sh` на стадии Verification открывает запись только для путей вида
`tests/**`, `test/**`, `spec/**` (включая вложенные). Но собственная конвенция этого
репозитория для тестов enforcement-слоя — **плоские** файлы `sdx/hooks/test-*.sh` рядом с
проверяемым скриптом, без каталога `tests/`.

Следствие воспроизведено на живой сессии: агенту `qa`, писавшему интеграционный тест
`sdx/hooks/test-integration-stage-lifecycle.sh` на этапе Verification, `Write` был заблокирован
как «код заморожен до гейта Execution», и файл пришлось создавать через `Bash`-heredoc —
технически легитимный путь (вне матчера `Write|Edit|MultiEdit`), но это ровно тот молчаливый
обход хука, который протокол запрещает (прецедент BUG-006).

Мета-проект — не единственный пострадавший: со-локованные тесты (`foo_test.go`, `*_test.py`
рядом с исходником) — распространённая конвенция, и per-project `stage-gate.allow` покрывает
их только если проект о нём знает.

## Рекомендация

Либо расширить встроенный allow-паттерн Verification на общеупотребимые формы со-локованных
тестов (`*/test-*.sh`, `*_test.*`, `test_*.*`), либо развернуть соответствующее правило в
`.claude/sdx/stage-gate.allow` этого репозитория шаблоном `/sdx:init`. Первое лечит класс
проблемы, второе — только текущий проект.
