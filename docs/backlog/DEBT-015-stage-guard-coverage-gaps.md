# DEBT-015. Пробелы тестового покрытия нового stage-enforcement

## Суть

Сессия закрытия DEBT-001 оставила два места, где мутация кода **не роняет ни одного теста** —
то есть поведение держится на честном слове, а не на регрессионной защите:

1. **Карве-аут первичного создания в `stage-write-guard.sh`** — ветка «файла ещё нет → хук
   пропускает». Удаление карве-аута не приводит к падению ни одного из 24 сценариев сьюта.
2. **`next`-путь баннера `SDX-OUTDATED`** — существующий тест подтверждает, что помеченный
   артефакт продолжает засчитываться как доказательство для `retrack`, но аналогичного
   сценария для `next` нет, хотя ревью проверило это поведение вручную и оно является частью
   заявленного контракта («баннер не блокирует гейты»).

Оба пробела найдены мутационным тестированием на 6-м проходе fresh-eyes ревью и сознательно
не закрыты в сессии-источнике, чтобы не растягивать её дальше.

## Рекомендация

Добавить два сценария в `sdx/hooks/test-stage-write-guard.sh` и `test-sdx-stage.sh`
соответственно; критерий готовности — мутация (удаление карве-аута; удаление баннер-нейтрального
поведения) роняет ровно новый сценарий. Работа на один заход, хорошо ложится в патч-трек.
