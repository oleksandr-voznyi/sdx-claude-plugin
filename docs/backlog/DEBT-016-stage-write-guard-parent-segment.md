# DEBT-016. `stage-write-guard.sh` не разрешает сегмент `..` в пути

## Суть

Deny-хук сравнивает путь операции с путём `session_state.json` **точно**, предварительно
нормализуя только схлопывание `/./` и повторных слэшей. Сегмент `..` не разрешается: путь вида
`/proj/.claude/sessions/x/../x/session_state.json` указывает на тот же файл, но точного
совпадения не даёт, и хук пропустит правку.

Практическая достижимость низкая — контракт инструментов Claude Code требует абсолютный путь,
и модель не строит пути с относительными сегментами без причины. Но это неустранённая
техническая дыра в компоненте, чья единственная задача — не пропускать правку.

## Рекомендация

Разрешать путь через `realpath`/`readlink -f` с фолбэком на текущую нормализацию, если утилита
недоступна (хук обязан оставаться работоспособным в минимальном окружении, как и при отсутствии
`jq`). Учесть, что файл может не существовать — `realpath -m`/`--canonicalize-missing`.
Закрывается тестом на `..`-сегмент по образцу существующего теста на `/./`.
