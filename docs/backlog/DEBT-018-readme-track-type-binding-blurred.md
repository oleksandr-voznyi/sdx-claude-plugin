# DEBT-018. Столбец «Типы сессий» в README стирает жёсткость привязки типа к треку `doc`

## Суть
`README.md` и `README.en.md` показывают типы сессий столбцом таблицы одинаково для всех
четырёх треков. Из такой подачи не видно ключевого различия, зафиксированного в REQ-DOC-7:
для `patch`/`standard`/`full` тип — лишь стартовая гипотеза, а трек выбирается триажем и
адаптивен; для `doc` связь `type → track` жёсткая 1:1 и без диалога.

## Рекомендация
Полстроки в оба README: пометить, что четыре типа трека `doc` привязаны к нему
безусловно, в отличие от триажа остальных треков.
