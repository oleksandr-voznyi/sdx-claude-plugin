# DEBT-019. doc-специфика пп. 2/3 Closeout есть в `protocol.md`, но не в печатающем чек-лист `archive.md`

## Суть
`sdx/protocol.md` описывает, что на треке `doc` пункты 2 (перенос дельт в `docs/specs`/
`docs/designs`) и 3 (архив `PLAN.md`) чек-листа закрытия — как правило `(n/a)`, с явным
исключением для сессий, меняющих сам протокол. Пункт 4 продублирован в обоих местах, а
уточнения к пп. 2/3 — только в протоколе. Печатает же чек-лист пользователю `commands/
archive.md`, то есть в момент реального Closeout doc-сессии эта информация не на виду.

## Рекомендация
Продублировать doc-оговорки пп. 2/3 в `commands/archive.md` (как уже сделано для п.4),
либо — системнее — убрать дублирование текста чек-листа между двумя файлами вовсе,
оставив в `archive.md` отсылку к протоколу. Второй вариант устраняет целый класс дрейфа,
но меняет структуру команды.
