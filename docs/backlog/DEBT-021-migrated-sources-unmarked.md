# DEBT-021. Уже мигрированные вручную источники этого репозитория не помечены `SDX-MIGRATED`

## Суть
Команда `/sdx:reconcile` считает идемпотентность повторного скана обеспеченной маркером
`<!-- SDX-MIGRATED: → <ID> (<дата>) -->` рядом с перенесённым фрагментом. Но в самом
репозитории SDX миграции выполнялись вручную ДО появления команды и маркеров не оставили:
находки `docs/audit-2026-07-01-recommendations.md` перенесены в `docs/backlog/` без пометок,
roadmap-заметки — тоже (`DEBT-008`).

Следствие: первый реальный прогон `/sdx:reconcile` в этом репозитории защищён от дублей
только прозаической дедупликацией по смыслу (шаг 5), без механической защиты маркерами.
На чужих проектах проблемы нет — там маркеры проставит сама команда.

## Рекомендация
Разово проставить маркеры `SDX-MIGRATED` в уже мигрированных источниках репозитория —
проще всего первым же прогоном `/sdx:reconcile --scan-only`, сверив его находки с
существующими записями бэклога и проставив маркеры по факту. Работа на один заход,
после неё повторные сканы этого репозитория становятся механически идемпотентными.

## Резолюция
Закрыто сессией `fw-reconcile-debt-20260720` (версия плагина 1.5.1). В
`docs/audit-2026-07-01-recommendations.md` проставлены 23 маркера
`<!-- SDX-MIGRATED: → <ID> (2026-07-19) -->` — по одному рядом с каждой находкой
`A1`–`A6`, `B1`–`B4`, `C1`–`C7`, `D1`–`D2`, `E1`–`E4`. Дата в маркере — дата фактической
миграции (сессия `fw-backlog-20260719`), а не дата расстановки: маркер документирует, когда
фрагмент был перенесён, а не когда это заметили.

Маппинг взят из полей `source` записей `docs/backlog/` и независимо перепроверен
fresh-eyes ревьюером: 23/23 совпадений, расхождений нет; множество находок файла в
точности равно множеству записей с `source: audit-2026-07-01 (…)`. Исходный текст находок
не изменён — дифф файла состоит из чистых вставок (23/0), файл остаётся историческим
снапшотом ревизии.

Повторный скан этого репозитория теперь отсекает уже мигрированное механически (шаг 4
команды), а не прозаической дедупликацией (шаг 5). Бандл `.sdx/bundles/upgrade_2026-06-27/`
маркерами сознательно НЕ помечен — по правилу, введённому `DEBT-022`: бандлы сканируются,
но не помечаются.

Сознательная не-цель: маркер уровня файла (одна пометка «мигрирован целиком» вместо N
пофрагментных) не вводился — это новая синтаксическая сущность в контракте команды ради
экономии строк в одном историческом файле.
