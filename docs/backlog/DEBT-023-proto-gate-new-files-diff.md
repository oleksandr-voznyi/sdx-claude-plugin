# DEBT-023. Гейт `/sdx:proto` не показывает содержимое новых файлов прототипа

## Суть
На гейте принятия/отклонения (`commands/proto.md`, шаг 4) дифф выводится только для
tracked-файлов (`git diff "$base_rev"`); для новых (untracked) файлов показываются лишь путь
и число строк. При этом REQ-VIBE-4 требует ЛИЧНОГО продуктового суждения пользователя, а
критерий приёмки 4 говорит про «список фактически тронутых/созданных файлов прототипа (и дифф)».

В типичном vibe-прототипе основная масса работы — именно новые файлы, и их содержимое
на гейте не предъявляется. Решение согласовано с дизайном (это документированный выбор,
а не расхождение), но против SPEC-критерия читается как сужение. Квитировано пользователем
на Verification сессии `feat-vibe-track-20260720`.

## Рекомендация
Показывать содержимое новых файлов на гейте — с тем же порогом усечения, что и для tracked-диффа
(`~400 строк → --stat + первые 200 строк + предложение развернуть`). Технически это
`git diff --no-index /dev/null <path>` по списку `$tmp_new` либо прямое чтение файлов.
