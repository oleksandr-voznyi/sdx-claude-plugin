# DEBT-025. Ручной прогон трека `vibe` не выполнен — покрытие ключевых критериев видимое

## Суть
Автотесты трека `vibe` покрывают матрицу состояний и два хука. Критерии приёмки 1, 3, 4, 8
и защита цикла `/sdx:archive` от зацикливания (критерий 5) не имеют исполняемого покрытия
по построению — они живут в прозе инструкций команд. Компенсация — `MANUAL_TEST.md`
(7 блоков A–G, 21 атомарный шаг), заархивированный вместе с сессией.

На момент закрытия сессии `feat-vibe-track-20260720` ни один из 21 шага не исполнен: поля
«Фактический результат» пусты. Покрытие этих критериев — «видимое» (apparent), не доказанное.
Квитировано пользователем: полноценный прогон требует реальной vibe-сессии на переустановленном
плагине, что естественнее делать после мёржа и `/plugin marketplace update sdx`.

Особого внимания требует блок A — он прямо проверяет найденный на Discovery риск бесконечного
цикла no-op итераций в `commands/archive.md`.

## Рекомендация
После обновления плагина завести реальную сессию `/sdx:start proto "<небольшая цель>"` и пройти
`MANUAL_TEST.md` (копия — в `docs/history/plans/feat-vibe-track-20260720.md` соседствующей
сессии; сам файл удалён вместе с каталогом сессии, восстановим из git-истории ветки).
Отдельно проверить блок B шаг 11 — названную границу: файл, созданный пользователем ПОСЛЕ старта
попытки, попадает в список отката.
