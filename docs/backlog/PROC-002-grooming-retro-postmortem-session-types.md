# PROC-002. Груминг / ретроспектива / постмортем как типы сессий

## Суть
Работа над самим бэклогом (пересмотр приоритетов, актуализация статусов, разбор
накопленного) сейчас не имеет своего типа сессии и трека — делается «по наитию» вне процесса.
Аналогично отсутствуют ретроспектива (периодический разбор завершённых сессий → выводы в
бэклог) и постмортем (разбор инцидента → корневая причина → действия).

## Рекомендация
Новые типы сессий `grooming` / `retro` / `postmortem` на общем лёгком
doc-only треке (Discovery-lite → Update → Closeout, без Execution-кода); шаблоны артефактов
(`retro.md`, `postmortem.md`); выходы всех трёх — записи бэклога. Зависит от E2. ADR.
