# PROC-005. Авторежим: смягчение пользовательских гейтов по запросу (фидбек 2026-07-19)

## Суть
Для тривиальных задач подтверждение каждого гейта дороже, чем одна проверка
перед закрытием (+ретрек при необходимости). Нужен opt-in авторежим: задача поставлена,
развилки согласованы на Discovery (там же принимается решение об авторежиме) — далее всё
автоматом до Closeout или ручного тестирования. Несущественные уточнения по ходу —
принимать дефолты и строго информировать о них перед закрытием; остановка — только на
«дорогих» и критичных развилках, где высока цена ошибочного решения.

## Рекомендация
Поле `gate_mode: interactive|auto` в `session_state.json`
(устанавливается на Discovery-гейте или `/sdx:start --auto`); прозаические гейты команд
(`next`, аппрувы Spec/Design/Plan, квитирование WARN) при `auto` принимают дефолты с
обязательной записью в `auto_decisions`-блок `change_note.md`; перед Closeout —
обязательный дисклоуз всех принятых дефолтов одним блоком (единственная точка
подтверждения). Жёсткая рубрика стоп-развилок, которые авторежим НЕ пропускает:
коллизии триады (CLAUDE.md §4 — без исключений), FAIL верификации, изменения
внешних контрактов/схем данных, деструктив, эскалация трека, территория prod-guard.
Детерминированные хуки не меняются. Связано с C3 (обязательность финальной проверки
на лёгких треках становится критичной) и E3 (общий механизм отложенного дисклоуза). ADR.

## Резолюция
Закрыто (сессия `fw-auto-gates-20260719`, 2026-07-19) — `gate_mode` в state, дефолты с
фиксацией в `auto_decisions.md`, стоп-рубрика, дисклоуз на входе в Closeout; ADR-014,
`docs/specs/gate-mode-auto.md`.
