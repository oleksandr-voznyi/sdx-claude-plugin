# PROC-007. Активные рекомендации по улучшению в ходе реальных сессий

## Суть
Трение фреймворка (обходы хуков, ложные срабатывания, лишняя церемония) обнаруживается в
реальных сессиях, но молча обходится и теряется — прецедент BUG-006: stage-gate на Windows
обходили через shell вместо репорта. Нужна явная инструкция оркестратору активно озвучивать
замеченные возможности улучшения и направлять их в бэклог.

## Рекомендация
Секция «Непрерывное улучшение» в `sdx/protocol.md`: столкнулся с трением → сообщи пользователю
1–2 строками и предложи `/sdx:backlog add` (или сессию доработки для мета-репозитория);
молчаливый обход запрещён. В авторежиме — не гейт: озвучивается без остановки.
