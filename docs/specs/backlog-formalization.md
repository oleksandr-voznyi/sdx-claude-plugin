# Спецификация: трекаемый бэклог проекта (docs/backlog/)

**Источник:** сессия `fw-backlog-20260719` (находка E2 аудита 2026-07-01 → запись `PROC-003`), ADR-015.

## Назначение
Бэклог проекта — постоянный трекаемый артефакт per-project слоя SDX, заменяющий
неструктурированное ведение находок (файл аудита + gitignored-roadmap). Обеспечивает:
адресуемость записей по ID, трассировку в документации, машиночитаемость для будущей
портфельной интеграции, планирование волн и невыпадение отложенных решений из процесса.

## Требования

### REQ-BL-1. Формат хранилища
`docs/backlog/` — одна запись = один файл `<ID>-<slug>.md`:
- YAML frontmatter (обязательные поля): `id`, `type` (`feat|bug|debt|idea|proc`),
  `status` (`open|in-progress|closed|deferred`), `priority` (`high|normal|low`),
  `wave` (номер волны или `null`), `source` (происхождение записи), `session`
  (сессия, закрывшая/ведущая запись, или `null`), `links` (связанные ADR/записи).
- Тело: `## Суть`, `## Рекомендация`; у закрытых/отложенных — `## Резолюция`.
- Slug — короткий английский kebab-case; язык тела — русский (языковая политика §1).

### REQ-BL-2. Идентификаторы
Префикс по типу: `FEAT-` / `BUG-` / `DEBT-` / `IDEA-` / `PROC-`; сквозная нумерация
внутри префикса (`NNN`, ведущие нули). ID неизменяем после создания; ре-классификация —
новая запись со ссылкой на старую (инвариант ADR-015).

### REQ-BL-3. Индекс
`docs/backlog/README.md`: описание схемы + таблица «Открытые» (open/in-progress/deferred;
сортировка: in-progress → open по priority, затем wave → deferred) и таблица «Закрытые»
(ID, название, сессия закрытия). Индекс обновляется при любом изменении состава/статусов.

### REQ-BL-4. Команда /sdx:backlog
Режимы: без аргументов — список; `--status/--type/--wave` — фильтры; `<ID>` — деталь;
`add` — создание записи (интервью, автономер, обновление индекса). Команда не требует
активной сессии; пишущий режим — только `add` (и явно запрошенная ручная правка статуса).

### REQ-BL-5. Closeout-интеграция
Пункт 4 Closeout-чек-листа (глобальный лог) расширен актуализацией бэклога: закрытые
сессией записи → `status: closed` + `session`; отложенные решения и неквитированные
WARN-находки Verification → новые `DEBT-`/`IDEA-`записи; индекс обновлён. Нумерация
пунктов чек-листа и инвариантов archive-verify при этом НЕ меняется (ADR-015).

### REQ-BL-6. Инициализация
`/sdx:init` создаёт `docs/backlog/` с пустым индексом из шаблона плагина
(`sdx/templates/backlog-readme.md`), если каталога ещё нет.

## Миграция (выполнена в fw-backlog-20260719)
Все 23 находки A*–E* файла `docs/audit-2026-07-01-recommendations.md` перенесены в
`docs/backlog/` с сохранением статусов и содержимого (маппинг старых ID — в поле `source`);
файл аудита оставлен историческим снапшотом с баннером-указателем и не обновляется.

## Границы
- Enforcement-слой не затронут: бэклог — прозаический слой (`docs/**` открыт stage-gate всегда).
- Схема frontmatter расширяется только аддитивно; изменения/удаления полей — новый ADR.
