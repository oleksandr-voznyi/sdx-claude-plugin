# Спецификация: авторежим гейтов (gate_mode) + обязательная лёгкая верификация patch

Сессия: `fw-auto-gates-20260719` (feature, standard). Источник: бэклог E4 + C3
(`docs/audit-2026-07-01-recommendations.md`, фидбек пользователя 2026-07-19).
Решение и инварианты: ADR-014 (`docs/DECISIONS.md`).

## Требования (Delta-формат)

- [ADDED] **REQ-AUTO-1 (opt-in per-session).** Пользователь может перевести сессию в авторежим
  прохождения прозаических гейтов: поле `gate_mode: "interactive" | "auto"` в
  `session_state.json` (default `interactive`; отсутствие поля = `interactive`). Включение —
  `/sdx:start --auto` либо согласие на предложение оркестратора на Discovery/Change-гейте;
  возврат в `interactive` — в любой момент по слову пользователя.
- [ADDED] **REQ-AUTO-2 (дефолты с фиксацией).** При `gate_mode: "auto"` прозаические
  подтверждения (аппрувы артефактов, переходы `/sdx:next`, несущественные уточнения)
  принимаются дефолтом без остановки; каждый дефолт фиксируется строкой
  `развилка → выбранный дефолт → почему несущественно` в трек-независимом файле сессии
  `auto_decisions.md`, версионируемом как обычный артефакт.
- [ADDED] **REQ-AUTO-3 (стоп-рубрика).** Авторежим обязан остановиться на: коллизиях триады
  (без исключений), FAIL верификации / красном тест-поле, изменениях внешних контрактов
  (API, схемы данных, форматы, миграции), деструктиве и территории prod-guard, эскалации
  трека, шагах `/sdx:manual`. Критерий по умолчанию: цена отката ошибочного дефолта выше
  цены остановки → стоп. Список — закрытый; сокращение — новый ADR.
- [ADDED] **REQ-AUTO-4 (дисклоуз).** Единственная обязательная точка подтверждения
  авторежима — гейт входа в Closeout (`/sdx:archive`, до начала чек-листа):
  `auto_decisions.md` выводится одним блоком (автоквитированные WARN — первыми), запрашивается
  одно явное подтверждение; без него чек-лист не начинается. Несогласие →
  `/sdx:backtrack`/`/sdx:retrack`.
- [ADDED] **REQ-AUTO-5 (enforcement не смягчается).** Детерминированные хуки (stage-gate,
  stop-gate, prod-guard, archive-verify) не читают `gate_mode` и работают одинаково в обоих
  режимах.
- [ADDED] **REQ-AUTO-6 (эскалация сбрасывает авто).** Эскалация трека через `/sdx:retrack`
  переводит `gate_mode` в `interactive`; деэскалация режим не меняет.
- [MODIFIED] **REQ-WARN-1 (отложенное квитирование).** Инвариант ADR-004 «WARN требует явного
  квитирования пользователем» уточнён: при `gate_mode: "auto"` квитирование отложенное —
  WARN автоквитируются с записью в `auto_decisions.md` и предъявляются первым блоком
  дисклоуза (REQ-AUTO-4). FAIL блокирует в любом режиме.
- [MODIFIED] **REQ-PATCHV-1 (верификация patch обязательна).** На треке `patch` Verification —
  обязательный активный этап в лёгком объёме, независимо от `gate_mode`: зелёный
  регрессионный тест + fresh-eyes `reviewer` (Task, контракт изоляции ADR-004) суженно против
  `change_note.md` (оси корректности и когерентности); субагент `qa` не вызывается.
- [ADDED] **REQ-PATCHV-2 (гейт закрытия).** `/sdx:archive` на треке `patch` не начинает
  чек-лист без существующего `verification_report.md` без находок FAIL.

## Реализация

Только прозаический слой: `sdx/protocol.md` (схема состояния, раздел «Авторежим гейтов»,
таблица треков, гейт входа в Closeout), `commands/{start,next,verify,archive,retrack}.md`,
`CLAUDE.md` §3/§4. Хук-скрипты и их тесты не менялись (регрессия 54/54).
