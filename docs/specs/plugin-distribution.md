# SPEC: Дистрибуция SDX как плагина Claude Code

Статус: действующая (сессия fw-plugin-20260719, 2026-07-19). Заменяет модель развертывания «копировать `.claude/` в каждый проект» (CLAUDE.md §6 до ADR-013).

## Контекст
SDD-процесс нужен в нескольких проектах на одном сервере. Тиражирование файлов фреймворка по проектам порождает дрейф версий и ручную синхронизацию правок.

## Требования

- [ADDED] **REQ-PLUGIN-1 (единая установка).** Фреймворк устанавливается один раз на машину как плагин Claude Code (scope `user`) и доступен во всех проектах: команды `/sdx:*`, субагенты ролей и enforcement-хуки поставляются плагином, а не файлами проекта.
- [ADDED] **REQ-PLUGIN-2 (неизменный UX).** Пространство имён команд сохраняется (`/sdx:start`, `/sdx:next`, …): имя плагина `sdx` даёт тот же неймспейс, что и прежний каталог `.claude/commands/sdx/`.
- [ADDED] **REQ-PLUGIN-3 (per-project слой).** В целевом проекте живут только его данные: `docs/` (specs/designs/history), артефакты сессий `.claude/sessions/<id>/`, конфиги enforcement `.claude/sdx/{prod-guard.conf, stage-gate.allow, verify-cmd.sh}` и targeted-паттерны `.gitignore`. Разворачивается командой `/sdx:init` из шаблонов плагина; существующие файлы не перезаписываются.
- [ADDED] **REQ-PLUGIN-4 (safe-by-default при глобальной установке).** Хуки плагина активны во всех проектах пользователя, поэтому обязаны оставаться no-op вне веток `sdx/<id>` и при отсутствии per-project конфигов (свойство подтверждается тест-сьютом хуков).
- [ADDED] **REQ-PLUGIN-5 (самодистрибуция).** Репозиторий фреймворка сам является и плагином (root-as-plugin), и marketplace: установка — `/plugin marketplace add <репо>` → `/plugin install sdx@sdx`. Мета-репозиторий разрабатывается через этот же установленный плагин (догфудинг).
- [ADDED] **REQ-PLUGIN-6 (миграция legacy-проектов).** `/sdx:init` детектирует старые локальные копии фреймворка (`.claude/commands/sdx/`, SDX-агенты в `.claude/agents/`) и предлагает удалить их во избежание дублей команд; per-project файлы при этом не трогаются.

## Критерии приёмки
- Плагин ставится из локального пути и из git-URL; `claude plugin list` показывает `sdx@sdx` enabled (scope user).
- В постороннем проекте без файлов фреймворка `/sdx:status` исполняется и корректно сообщает об отсутствии сессии.
- В проекте с активной SDX-сессией stage-gate из плагина блокирует запись в код вне Execution/Deployment.
- Прогон тест-сьюта хуков из состава плагина: все зелёные.
