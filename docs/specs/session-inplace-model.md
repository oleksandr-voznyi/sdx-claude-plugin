# Business Specification: Сессия = ветка в основном рабочем дереве (отказ от worktree, один CLI)

Источник: сессия `fw-session-inplace-20260705` (трек standard, change_note), запрос пользователя 2026-07-05.
Авторитетная запись решения — **ADR-012** в `docs/DECISIONS.md`. Пересматривает worktree-часть
`docs/specs/session-worktree-model.md` (REQ-WT-1/3/4/5).

## Проблема
Worktree-модель (ADR-009) сделала процесс операционно тяжёлым: `/sdx:start` заканчивался
хендоффом «откройте новый CLI в `.sdx/worktrees/<id>/`», Closeout требовал двух CLI-инстансов.
Непрерывность разработки в одном CLI ломалась на каждом старте и закрытии сессии; цена
физической изоляции параллельных «грязных» деревьев (REQ-WT-3) оказалась выше её пользы.

## Требования
- [ADDED] **REQ-INPLACE-1**: Полный жизненный цикл сессии (от `/sdx:start` до `/sdx:archive`
  включительно) проходит в ОДНОМ CLI-инстансе, запущенном в корне репозитория. Ни одна команда
  `/sdx:*` не инструктирует открыть другой CLI в штатном флоу.
- [ADDED] **REQ-INPLACE-2**: Сессия ведётся на ветке `sdx/<id>`, checkout'нутой в основном
  рабочем дереве. `/sdx:start` = `git checkout -b sdx/<id>`; `/sdx:switch` = `git checkout
  sdx/<id>`. Обе операции требуют чистого рабочего дерева; при грязном — стоп с инструкцией
  пользователю (авто-коммит чужих изменений запрещён — ядро REQ-WT-2 сохранено).
- [ADDED] **REQ-INPLACE-3**: Enforcement-хуки резолвят активную сессию по имени текущей ветки
  (`sdx/<id>`); переход на in-place-модель не требует правок логики хуков.
- [MODIFIED] Closeout — однофазный чек-лист в одном CLI: шаги 1–5 на ветке сессии (включая
  `git rm -r` каталога сессии коммитом ДО мёржа — вариант A), затем checkout основной ветки,
  `merge --no-ff`, `archive-verify.sh`. Инварианты 1/5/6 и REQ-SESS-1..4 действуют дословно.
- [REMOVED] REQ-WT-1/3/4/5 (создание worktree, физическая изоляция параллельных сессий,
  освобождение worktree) — потеряли предмет. Возврат к физической изоляции — новый ADR.

## Совместимость
- `archive-verify.sh` сохраняет условное `git worktree remove --force` для legacy-сессий,
  созданных до ADR-012; без worktree блок — no-op.
- Паттерн `.sdx/worktrees/` в `.gitignore` существующих проектов безвреден и может оставаться;
  из шаблона `/sdx:init` он исключён.

## Критерии приемки (проверены сессией-догфудом fw-session-inplace-20260705)
- [x] Полный прогон start → change → execution → verification → closeout в одном CLI, без хендоффов.
- [x] Тест-сьюты хуков зелёные без правок логики: archive-verify 18/18, stop-gate 13/13,
      stage-gate 8/8, default-branch 6/6.
- [x] В командах `/sdx:*` не осталось инструкций «откройте новый CLI» для штатного флоу.
- [x] REQ-SESS-1..4, вариант A и ADR-010 сохранены дословно.
