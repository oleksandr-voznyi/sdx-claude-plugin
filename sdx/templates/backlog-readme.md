# Бэклог проекта

Трекаемый бэклог (ADR-015 SDX): одна запись = один файл `<ID>-<slug>.md` с машиночитаемым
YAML frontmatter (`id`, `type`, `status`, `priority`, `wave`, `source`, `session`, `links`)
и телом `## Суть` / `## Рекомендация` (для закрытых — `## Резолюция`).

Префиксы ID по типам: `FEAT-` (новая функциональность), `BUG-` (дефекты/противоречия),
`DEBT-` (техдолг/дрейф/недоспецификация), `IDEA-` (roadmap-идеи), `PROC-` (процессные
изменения). Статусы: `open | in-progress | closed | deferred`. Поле `wave` — номер волны
планирования (или `null`).

Просмотр, фильтры и создание записей — команда `/sdx:backlog`. Статусы актуализируются
на Closeout каждой сессии (пункт 4 чек-листа): закрытые записи получают `status: closed`
и `session`, отложенные решения и неквитированные WARN становятся новыми `DEBT-`/`IDEA-`записями.
