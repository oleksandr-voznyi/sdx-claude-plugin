## SDX (Spec-Driven Development)

Проект разрабатывается по методологии SDX через одноимённый плагин Claude Code (команды `/sdx:*`).

- **Жизненный цикл**: работа ведётся в изолированных сессиях (`/sdx:start`), сессия = git-ветка `sdx/<id>`, артефакты — в `.claude/sessions/<id>/`. Треки церемонии: `patch` / `standard` / `full` (адаптивны, `/sdx:retrack`). Закрытие — строго по чек-листу `/sdx:archive`.
- **Источники правды (триада)**: [SPEC] ↔ [DESIGN] ↔ [Codebase] (постоянные документы: `docs/specs/`, `docs/designs/`). При обнаружении логического противоречия между источниками — ОСТАНОВИСЬ и запроси решение у пользователя, не выбирай сторону молча.
- **Языковая политика**: общение в чате, спецификации и пользовательская документация — русский; код (имена, комментарии) и техдокументация API (OpenAPI/Swagger) — английский.
- **Роли**: оркестрация — основная сессия (Session Manager, команды `/sdx:*`); специализированные субагенты плагина (`ba`, `architect`, `lead-dev`, `developer`, `qa`, `reviewer`, `tech-writer`, `devops`) вызываются только из основной сессии через Task.
